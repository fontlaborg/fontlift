Change this so that it is (note: -c and -i are mutually exclusive): 

```
  -c, --copy
          Copy the font file to the fonts folder and install (default)
  -i, --inplace
          Keep the font file in place but install it
  -a, --admin
          Install at system level (all users, requires admin)
      --dry-run
          Print intended actions without mutating fonts
  -V  --no-validate
```

Also add single-lettet flags in other CLI options. 

Make sure that the app doesn’t fail on simple tasks but instead tries to recover itself, for example elevates to admin mode, and gives informative messages. 

Use ./tests/Lato-Thin.ttf for testing. 

For example: 

```
$ fontlift uninstall /Users/adam/Library/Fonts/Lato-Thin.ttf
Uninstalling font from path: /Users/adam/Library/Fonts/Lato-Thin.ttf
✅ Successfully uninstalled font (user-level)
$ fontlift remove /Users/adam/Library/Fonts/Lato-Thin.ttf
Removing font from path: /Users/adam/Library/Fonts/Lato-Thin.ttf
❌ Error: Font registration failed: Core Text failed to unregister font /Users/adam/Library/Fonts/Lato-Thin.ttf: The file is not registered in the specified scope.
→ Suggestion: Try restarting your system or using administrator/sudo privileges
```

It should remove the font file even if it is no longer registered instead of failing, but may issue a warning. 

When using --admin and --name and the font is not in the system scope but is in the user scope, the app shouldn't fail. 

Situations like 'fontlift uninstall -n "Lato-Thin"' should not fail if Lato-Thin is not installed, should just issue warning. Failure to install is an error. Failure to uninstall if the font is installed is an error. But failure to uninstall because the font was not install should be a warning.
